

------

## 1. 项目愿景 (Project Vision)

打造一个专属于校园内部的、具有时空感的匿名社交工具。通过“纸飞机”这一载体，让学生在特定的校园地点（如图书馆、操场、食堂）留下情感或信息碎片，由后续到达该地点的学生开启，产生一种“时空交错”的互动体验。

------

## 2. 用户角色 (User Personas)

- **投掷者 (Thrower)：** 希望在特定地点表达情绪、分享信息或寻求帮助的学生。
- **拾取者 (Picker)：** 处于特定地点，通过“查看该地点飞机”来打发时间或获取信息的学生。
- **管理员 (Admin)：** 负责预设校园地点标签（如：教学楼 A、二食堂）及清理违规内容。

------

## 3. 功能需求 (Functional Requirements)

### 3.1 核心功能

| **功能模块**            | **子功能**   | **详细描述**                                                 |
| ----------------------- | ------------ | ------------------------------------------------------------ |
| **投掷飞机 (Throw)**    | **内容编辑** | 支持输入文字（限 200 字），可选投掷心情（如：开心、吐槽、求助）。 |
|                         | **地点选择** | 用户从后端预设的**列表**（如：图书馆、操场）中选择一个降落点。 |
|                         | **时效设置** | 设置飞机存在时间：2h、24h 或 48h，过期自动消失。             |
| **发现飞机 (Discover)** | **地点筛选** | 用户点击某个地点（如“二食堂”），系统展示当前降落在该处的飞机列表。 |
|                         | **雷达动效** | (视觉效果) 进入页面时展示一个简易的雷达扫描动画，随后弹出飞机。 |
| **开启与互动 (Action)** | **内容拆阅** | 点击飞机图标，展示精美的“信纸”弹窗。                         |
|                         | **匿名回应** | 拾取者可以留下简短回复，回复者自动获得一个随机生成的匿名代号（如：路过的学长）。 |
| **系统自动管理**        | **自动降解** | C# 后端定时检查 `ExpireTime`，将过期的飞机标记为不可见。     |

### 3.2 管理功能

- **地点配置：** 后端可灵活增加或删除校园内的地点标签。
- **违规处理：** 提供简单的“举报”按钮，被举报超过 3 次的飞机自动下线。

------

## 4. 技术架构 (Technical Stack)

- **前端 (Frontend)：** **Vue 3 (Vite)** + **Vant UI**（Vant 的弹出层和单元格组件非常适合移动端展示）。
- **后端 (Backend)：** **ASP.NET Core 8 Web API (C#)**。
- **数据库 (Database)：** **SQLite**（单文件存储，无需安装大型数据库软件，最适合初学者）。
- **部署：** 后端运行在本地或云服务器，前端通过 Axios 与后端通信。

## 5. 数据模型设计 (Data Model)

这是 C# 后端的核心模型定义，简洁但覆盖了所有业务：

| **实体名**  | **字段 (Field)** | **类型 (Type)** | **说明**                                     |
| ----------- | ---------------- | --------------- | -------------------------------------------- |
| **Plane**   | `Id`             | `Guid`          | 主键唯一标识                                 |
|             | `LocationTag`    | `String`        | 地点（如：图书馆）                           |
|             | `Content`        | `String`        | 飞机内容                                     |
|             | `Mood`           | `String`        | 情绪标签                                     |
|             | `CreateTime`     | `DateTime`      | 投掷时间                                     |
|             | `ExpireTime`     | `DateTime`      | 过期时间 ($ExpireTime = CreateTime + hours$) |
|             | `PickCount`      | `Int`           | 被查看次数                                   |
| **Comment** | `Id`             | `Guid`          | 评论 ID                                      |
|             | `PlaneId`        | `Guid`          | 关联的飞机 ID                                |
|             | `Reply`          | `String`        | 回复内容                                     |
|             | `NickName`       | `String`        | 随机生成的匿名名                             |

------

## 6. 核心业务流程 (Workflow)

1. **用户进入 Vue 首页：** 展示校园地点列表（例如：操场(5), 图书馆(12)）。
2. **投掷：** 用户点击“投掷”，选择“图书馆”，后端将一条记录写入 SQLite。
3. **拾取：** 另一位用户在图书馆感到无聊，点开“图书馆”标签，Vue 向后端请求 `GET /api/planes?location=图书馆`。
4. **互动：** 后端过滤掉 `DateTime.Now > ExpireTime` 的数据，返回活跃的飞机。

------

## 7. 界面原型建议 (UI Concept)

- **首页：** 采用平铺的卡片流，每个卡片是一个地点。
- **投掷页：** 一个大大的输入框，背景是一个若隐若现的纸飞机折纸背景。
- **拾取页：** 模仿“拆开信封”的过程，加入 Vue 的过渡动画 (Transition)。
