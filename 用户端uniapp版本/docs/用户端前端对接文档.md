## 2. 需要前端特别注意的点（避免“像缺接口”）
- `POST /api/planes/{id}/like` 现在必须登录，否则会 `401`。
- `POST /api/planes/{id}/recall`、`POST /api/planes/{id}/destroy` 必须登录且仅作者本人可操作，否则 `403`。
- `POST /api/planes` 虽未强制登录，但若不带用户 token，`CreatorUserId` 为空，这条投递不会出现在“我的投递”。
- `GET /api/planes/{id}`、`GET /api/planes/random` 只有在带用户 token 时才会记入“我的拾取”。

## 3. 基础约定
- Base URL：`{HOST}/api`
- 鉴权头：`Authorization: Bearer {accessToken}`
- 成功响应：直接返回业务对象，不包一层 `data`
- 失败响应：一般为 `{ "message": "..." }`
- 时间字段：ISO 8601 UTC 字符串（例如 `2026-04-08T10:00:00Z`）

## 4. 用户认证接口

### 4.1 注册
- `POST /api/user-auth/register`
- 鉴权：否
- 请求体：
```json
{
  "username": "张三",
  "studentId": "20230001",
  "phone": "13800000000",
  "password": "123456"
}
```
- 成功响应 `200`：
```json
{
  "userId": "uuid",
  "username": "张三",
  "phone": "13800000000",
  "studentId": "20230001"
}
```
- 可能错误：
1. `400` 参数不合法（用户名 1-12，学号 6-20 字母数字，手机号 11 位，密码 >= 6）
2. `409` 学号或手机号已注册

### 4.2 登录（学号或手机号）
- `POST /api/user-auth/login`
- 鉴权：否
- 请求体：
```json
{
  "credential": "20230001",
  "password": "123456"
}
```
- 成功响应 `200`：
```json
{
  "accessToken": "jwt",
  "refreshToken": "token",
  "expiresIn": 3600,
  "user": {
    "userId": "uuid",
    "username": "张三",
    "avatarUrl": "/uploads/avatars/xxx.png",
    "gender": "secret",
    "bio": ""
  }
}
```
- 可能错误：
1. `400` 账号/密码为空
2. `401` 账号或密码错误

### 4.3 刷新 token
- `POST /api/user-auth/refresh-token`
- 鉴权：否
- 请求体：
```json
{
  "refreshToken": "token"
}
```
- 成功响应：与登录响应一致
- 可能错误：`400`、`401`

### 4.4 获取当前用户
- `GET /api/user-auth/me`
- 鉴权：是
- 成功响应 `200`：
```json
{
  "userId": "uuid",
  "username": "张三",
  "avatarUrl": "",
  "gender": "secret",
  "bio": ""
}
```
- 可能错误：`401`

### 4.5 退出登录
- `POST /api/user-auth/logout`
- 鉴权：是
- 请求体可选：
```json
{
  "refreshToken": "token"
}
```
- 成功响应 `200`：
```json
{
  "message": "退出成功"
}
```
- 说明：不传 `refreshToken` 时，会撤销该用户当前所有未过期 refresh token。

## 5. 个人资料接口

### 5.1 获取资料
- `GET /api/users/me/profile`
- 鉴权：是
- 成功响应 `200`：
```json
{
  "username": "张三",
  "avatarUrl": "/uploads/avatars/xxx.png",
  "gender": "secret",
  "bio": "..."
}
```

### 5.2 更新资料
- `PUT /api/users/me/profile`
- 鉴权：是
- 请求体：
```json
{
  "username": "张三",
  "avatarUrl": "/uploads/avatars/xxx.png",
  "gender": "male",
  "bio": "..."
}
```
- 可能错误：
1. `400` 用户名长度不合法
2. `400` `gender` 非 `male/female/secret`
3. `400` `bio` 超长（>200）

### 5.3 上传头像
- `POST /api/users/me/avatar`
- 鉴权：是
- Content-Type：`multipart/form-data`
- 表单字段：`file`
- 成功响应：
```json
{
  "url": "/uploads/avatars/xxx.png"
}
```
- 可能错误：`400`（格式/大小不合法，单图最大 5MB）、`401`

## 6. “我的”模块接口

### 6.1 我的投递
- `GET /api/planes/mine/thrown`
- 鉴权：是
- Query：
1. `status=all|active|recalled|expired`（默认 `all`）
2. `keyword`（按 `content/locationTag` 模糊匹配）
3. `page`（默认 1）
4. `pageSize`（默认 20，最大 100）
- 成功响应：
```json
{
  "items": [
    {
      "id": "uuid",
      "locationTag": "图书馆",
      "content": "...",
      "mood": "calm",
      "createTime": "2026-04-08T10:00:00Z",
      "expireTime": "2026-04-09T10:00:00Z",
      "pickCount": 10,
      "likeCount": 3,
      "commentCount": 2,
      "status": "active",
      "isRecalled": false,
      "fueledAt": null,
      "pickedAt": null
    }
  ],
  "total": 1
}
```

### 6.2 我的续航
- `GET /api/planes/mine/fueled`
- 鉴权：是
- Query：`keyword/page/pageSize`
- 返回结构同上，`fueledAt` 有值

### 6.3 我的拾取
- `GET /api/planes/mine/picked`
- 鉴权：是
- Query：`keyword/page/pageSize`
- 返回结构同上，`pickedAt` 有值

## 9. 前端改造建议（最小改动）

### 9.1 全局请求封装
- 在 `uni.request` 与 `uni.uploadFile` 统一注入 `Authorization`。
- accessToken 过期后，用 refreshToken 调 `POST /api/user-auth/refresh-token`，刷新成功后重放请求。
- refresh 失败则清空会话并跳登录页。

### 9.2 登录注册与资料
- `registerAccount/loginAccount/logoutAccount` 从本地存储逻辑改为接口调用。
- 启动时调用 `/api/user-auth/me` 校验登录态。
- “个人资料页”改调 `/api/users/me/profile`、`/api/users/me/avatar`。

### 9.3 我的页
- “我的投递”改调 `/api/planes/mine/thrown`
- “我的续航”改调 `/api/planes/mine/fueled`
- “我的拾取”改调 `/api/planes/mine/picked`
- 废弃本地 `myPlaneIds/fueledPlaneIds/recalledPlaneIds` 作为主数据源（可保留短期缓存）

## 10. 推荐联调顺序
1. 注册 -> 登录 -> `me` -> 退出
2. 获取/修改 profile -> 上传 avatar
3. 投递飞机（带 token）-> 我的投递
4. 详情拾取/随机拾取 -> 我的拾取
5. 续航 -> 我的续航
6. 回收/销毁作者鉴权验证
