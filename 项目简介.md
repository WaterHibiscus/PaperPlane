## ✈️ 「纸飞机降落点」核心功能地图

### 1. 投掷模块：创作与定义 (The Launch)

这是用户输出内容的入口，重点在于**简单、快速、有仪式感**。

- **多模态内容录入：**
  - **文字信笺：** 支持 200 字以内的短文，适合随笔、吐槽或求助。
  - **情绪着色 (Mood Coloring)：** 用户选择心情（如：✨期待、🌧️忧郁、🔥热血），Vue 前端会根据心情改变纸飞机的背景颜色，SQL Server 同步记录颜色代码。
- **物理点指派：**
  - 用户从下拉列表或搜索框选择预设的校园地点（如：*二教 201、西区操场、明德湖畔*）。
- **延时起飞 (Optional)：**
  - 可以设置飞机在 5 分钟后才“降落”，模拟飞机在空中滑行的过程，增加期待感。

### 2. 发现模块：雷达与拾取 (The Discovery)

这是项目的灵魂，让用户产生“我就在现场”的共鸣。

- **校园热力看板：**
  - 首页展示校园地图的简图或地点列表，每个地点显示当前“存量”飞机数。
- **伪雷达扫描：**
  - 点击某个地点时，Vue 触发一个圆形的波纹扫描动画，模拟后端在 SQL Server 中检索数据的过程。
- **拆封仪式：**
  - 点击飞机图标，伴随“拆纸”音效或折纸展开的 CSS 动画，展示内容。

### 3. 互动模块：回声系统 (The Echo)

为了保护校园隐私并降低社交压力，互动采取**全匿名**模式。

- **匿名回声：**
  - 拾取者可以对飞机进行回复。系统会根据 SQL Server 预设的随机词库分配匿名 ID（如：*“正在吃面的学弟”*、*“路过的考研党”*）。
- **点赞续航 (Fuel Up)：**
  - 如果一只飞机被点赞，SQL Server 中的 `ExpireTime` 会自动增加（例如每赞一次加 1 小时），让优质内容停留更久。
- **投掷者反馈：**
  - 当有人拾取或回复时，投掷者在下次登录时会收到“你的飞机在图书馆被开启了”的系统提醒。

### 4. 自动化管理：系统自愈 (The Engine)

利用 C# 后台服务和 SQL Server 的特性，让项目看起来很“聪明”。

- **自动降解 (Auto-Shredder)：**
  - C# 后端运行一个 `BackgroundService`，每隔 10 分钟执行一次 SQL 命令：`UPDATE Planes SET IsDeleted = 1 WHERE ExpireTime < GETDATE()`。
- **敏感词拦截：**
  - 投掷时，后端会对 `Content` 进行过滤，确保校园社交环境的纯净。
- **地点动态配置：**
  - 管理员可以随时在后台增加“临时降落点”（如：*“草坪音乐会专项”*），前端 Vue 会自动刷新地点列表。
